# GeoTrade Radar 上云部署指南（阿里云/腾讯云轻量 · 公网 IP 直连 · 演示用）

> 目标：把网站和后台部署到一台云服务器，通过 `http://公网IP` 直接访问。
> 不涉及域名、备案、HTTPS（这些以后要正式对外时再加）。

---

## 0. 买服务器

- 在 **阿里云 / 腾讯云** 买一台 **轻量应用服务器**（最低配 2C2G 足够演示）。
- 镜像选 **Ubuntu 22.04 或 24.04**（纯系统，不要选「宝塔」镜像，以免 80 端口被占）。
- 记下 **公网 IP**，并设置好 root 密码（或 SSH 密钥）。

## 1. ⚠️ 放行端口（最容易漏，漏了外网就打不开）

进云厂商控制台 → 找到这台服务器的 **防火墙 / 安全组** → 添加一条规则：

| 协议 | 端口 | 来源 |
|------|------|------|
| TCP  | 80   | 0.0.0.0/0（所有） |

> 如果你打算用 4173 端口，就放行 4173。

## 2. 把代码上传到服务器

在**你自己电脑**的 PowerShell 里执行（Win10/11 自带 scp）：

```powershell
scp -r "C:\Users\大爷\Documents\geotrade-radar" root@公网IP:/opt/geotrade
```

首次会提示 `yes`，然后输入服务器 root 密码。
（如果代码在 GitHub，也可以登录服务器后 `git clone` 到 `/opt/geotrade`。）

> 可选：想让线上从**干净的用户库**开始（不带本地测试账号），上传后在服务器执行
> `rm -f /opt/geotrade/users.db`，程序首次启动会自动重建空库。

## 3. 登录服务器并一键部署

```bash
ssh root@公网IP
cd /opt/geotrade
sudo PORT=80 ADMIN_TOKEN='换成你的强口令' bash deploy/setup.sh
```

脚本会自动：装 Node 24 → 建 systemd 服务 → 启动 → 设为开机自启，
最后打印出访问地址。

## 4. 打开

- 网站：`http://公网IP`
- 后台：`http://公网IP/admin.html` → 输入你在上一步设置的口令

## 5. 日常运维

```bash
journalctl -u geotrade -f      # 看实时日志（排错用）
systemctl restart geotrade     # 重启
systemctl stop geotrade        # 停止
systemctl status geotrade      # 看运行状态
```

更新代码后：重新 `scp` 覆盖文件，再 `systemctl restart geotrade` 即可。

---

## 注意事项（演示阶段可接受，正式对外前请处理）

1. **口令别用 `test123`**：部署时务必把 `ADMIN_TOKEN` 换成强口令。
2. **目前是 HTTP 明文**：登录信息未加密，仅适合演示。正式对外建议加域名 + HTTPS（用 Caddy 自动签证书，需要时找我配）。
3. **进程以 root 运行**：演示够用；正式环境建议建独立用户运行并最小化权限。
4. **数据备份**：所有用户/会话都在 `/opt/geotrade/users.db` 这一个文件里，定期复制备份即可。
5. **80 端口被占**：若提示端口占用（自带 nginx/宝塔等），改用 `sudo PORT=4173 ADMIN_TOKEN='...' bash deploy/setup.sh`，并在安全组放行 4173，访问 `http://公网IP:4173`。
