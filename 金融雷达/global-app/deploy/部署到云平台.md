# GeoTrade Radar 上线部署指南（海外受众版）

本项目是**零依赖** Node 服务（用内置 `node:sqlite`，需 **Node ≥ 22.5**，镜像已用 Node 24）。
已为公网部署做好准备：监听地址、端口、数据目录均由环境变量控制，前端 API 走相对路径，换域名零改动。

> ⚠️ 数据源（GDELT / Yahoo / OFAC / UN Comtrade 等）多在境外，**服务器请放海外节点**，否则大量数据会降级。

---

## 关键环境变量

| 变量 | 说明 | 部署建议值 |
|------|------|-----------|
| `HOST` | 监听地址 | `0.0.0.0`（对外必填） |
| `PORT` | 端口 | 平台自动注入；本地默认 4173 |
| `DATA_DIR` | 用户库/缓存写入目录 | 指向**持久磁盘**（如 `/data`） |
| `ADMIN_TOKEN` | 管理后台口令(admin.html) | **设成强口令**（密钥，勿入库） |

---

## 方案一：Render（推荐，最省事）

仓库已含 `Dockerfile` 与 `render.yaml` 蓝图，连 GitHub 即可。

1. 注册 https://render.com → **New → Blueprint**。
2. 选择本 GitHub 仓库，Render 自动读取 `render.yaml` 创建 Web 服务 + 1GB 持久磁盘（挂载 `/data`）。
3. 在提示处填入 **`ADMIN_TOKEN`**（强口令）。
4. 点击部署，几分钟后获得 `https://<名字>.onrender.com`，自带 HTTPS。
5. 以后 `git push` 到 master 即自动重新部署。

说明：
- `plan: starter`（约 $7/月）常驻不休眠、含持久磁盘 → 后台实时数据同步不中断、注册用户不丢。
- 想先免费试：把 `render.yaml` 的 `plan` 改 `free` 并删掉 `disk` 段——但**免费档会休眠**（首次访问冷启动慢）且 `users.db` 在重新部署时重置。

## 方案二：Docker（任意主机 / Fly.io / Cloud Run）

```bash
# 构建
docker build -t geotrade-radar .

# 运行（持久化用户库到主机 ./data）
docker run -d --name geotrade -p 80:8080 \
  -e ADMIN_TOKEN='改成你的强口令' \
  -v "$(pwd)/data:/data" \
  geotrade-radar
```
访问 `http://<服务器IP>`。HTTPS 建议在前面加 Caddy/Nginx 或用平台自带证书。

---

## 方案三：云服务器 VPS（完全掌控）

适合 1 核轻量 VPS（Ubuntu/Debian）。上传项目后：

```bash
cd /opt/geotrade
sudo PORT=80 ADMIN_TOKEN='改成你的强口令' bash deploy/setup.sh
```
脚本自动：装 Node 24、用 systemd 守护（崩溃重启、开机自启）、`HOST=0.0.0.0`。
> 建议加 `DATA_DIR=/opt/geotrade/data` 把用户库与代码分离，便于备份/升级。
> 域名 + HTTPS 用 Caddy 最省事（自动签发证书）。

---

## 上线后检查

- `https://你的域名/api/health` 返回 `{"ok":true}`
- 注册一个账号能登录、进入各雷达
- 管理后台 `admin.html` 用 `ADMIN_TOKEN` 登录
- 若某雷达显示"演示/陈旧"：多为该数据源对服务器 IP 限流（Yahoo 等常见），属正常降级，可换节点或加缓存
